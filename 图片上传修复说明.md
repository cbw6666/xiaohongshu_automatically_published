# 小红书自动发布插件 - 图片上传功能修复说明

## 问题描述
插件在发布环节存在多个问题，经过分析发现需要修复以下几个方面：
1. 需要先点击"图文"按钮切换到正确的上传模式
2. 图片上传元素的选择器发生了变化
3. 标题、正文、标签输入的元素选择器和逻辑需要更新
4. 需要使用新的DOM结构来填写内容

## 问题定位

### 1. 图文按钮选择器
**新的正确选择器：**
```
#web > div > div > div > div.header > div.header-tabs > div:nth-child(3) > span
```

**原始选择器：**
```
#web > div.outarea.upload-c > div > div > div.header > div:nth-child(2) > span
```

### 2. 图片上传元素选择器
**新的正确选择器：**
```
#web > div > div > div > div.upload-content.hasBannerHeight > div.upload-wrapper > div > input
```

**原始选择器：**
```
#web > div > div > div > div.upload-content > div.upload-wrapper > div > input
```

**元素属性：**
```html
<input data-v-4506f808="" data-v-7cbccdb2-s="" class="upload-input" type="file" multiple="" accept=".jpg,.jpeg,.png,.webp">
```

### 3. 标题输入元素选择器
**新的正确选择器：**
```
#web > div > div > div > div > div.body > div.content > div.plugin.title-container > div > div > div.input > div.d-input-wrapper.d-inline-block.c-input_inner > div > input
```

**元素属性：**
```html
<input class="d-text" type="text" placeholder="填写标题会有更多赞哦～" value="123">
```

### 4. 正文编辑器选择器
**新的正确选择器：**
```
#web > div > div > div > div > div.body > div.content > div.plugin.editor-container > div > div > div.editor-container > div.editor-content > div > div
```

**内容结构：**
- 每一段内容使用 `<p>` 标签包装
- 空行使用 `<p><br class="ProseMirror-trailingBreak"></p>`
- 有内容的行使用 `<p>内容文本</p>`

### 5. 标签结构
**新的标签结构：**
```html
<p>
  <a class="tiptap-topic" data-topic='{"name":"蝴蝶号","id":"643a5b41000000002c0312c3"}' contenteditable="false">
    #蝴蝶号<span class="content-hide">[话题]#</span>
  </a>
</p>
```

**关键属性：**
- `data-topic`: JSON格式，包含标签名称和ID
- `contenteditable="false"`: 标签不可编辑
- 标签名称放在 `name` 字段中

## 修复方案

### 1. 多选择器兼容性策略
为了提高插件的稳定性和兼容性，实现了多选择器尝试机制：

#### 图文按钮选择器列表：
```javascript
const selectors = [
  '#web > div > div > div > div.header > div.header-tabs > div:nth-child(3) > span',  // 最新选择器
  '#web > div.outarea.upload-c > div > div > div.header > div:nth-child(2) > span',  // 原始选择器
  '#web > div > div > div > div.header > div:nth-child(2) > span',  // 兼容选择器
  '.header-tabs > div:nth-child(3) > span',  // 基于类名的选择器
  'span:contains("图文")'  // 基于文本内容的选择器
];
```

#### 图片上传选择器列表：
```javascript
const selectors = [
  '#web > div > div > div > div.upload-content.hasBannerHeight > div.upload-wrapper > div > input',  // 最新选择器
  '#web > div > div > div > div.upload-content > div.upload-wrapper > div > input',  // 之前的选择器
  '#web > div.outarea.upload-c > div > div > div.upload-content > div.upload-wrapper > div > input',  // 原始选择器
  'input[data-v-4506f808][data-v-7cbccdb2-s].upload-input[type="file"][multiple][accept*=".jpg"]',  // 基于属性的选择器
  '.upload-input[type="file"]',  // 基于类名的选择器
  'input[type="file"][multiple][accept*=".jpg"]',  // 通用文件输入框选择器
  'input[type="file"][accept*="image"]'  // 图片文件输入框选择器
];
```

#### 标题输入选择器列表：
```javascript
const titleSelectors = [
  '#web > div > div > div > div > div.body > div.content > div.plugin.title-container > div > div > div.input > div.d-input-wrapper.d-inline-block.c-input_inner > div > input',  // 最新选择器
  '.titleInput input',  // 通用选择器
  'input[placeholder*="标题"]',  // 基于占位符的选择器
  'input.d-text[type="text"]'  // 基于类名的选择器
];
```

#### 正文编辑器选择器列表：
```javascript
const editorSelectors = [
  '#web > div > div > div > div > div.body > div.content > div.plugin.editor-container > div > div > div.editor-container > div.editor-content > div > div',  // 最新选择器
  '#quillEditor > div',  // 原始选择器
  '.editor-content > div > div',  // 通用选择器
  '[data-placeholder*="正文"]'  // 基于属性的选择器
];
```

### 3. 新增功能

#### 调试工具
在插件界面中添加了"调试上传元素"按钮，可以：
- 检测当前页面中所有可能的上传元素
- 显示每个选择器的匹配结果
- 列出页面中所有文件输入框及其属性
- 帮助用户快速定位问题

#### 错误处理增强
- 添加了详细的错误信息和成功日志
- 记录使用的选择器信息
- 提供更好的调试日志
- 找不到元素时不会直接退出，而是记录错误并继续执行

#### 内容填写优化
- **智能DOM结构生成**：直接创建p标签和a标签，而不是模拟用户输入
- **更快的填写速度**：避免了逐字符输入的延迟
- **更稳定的标签处理**：使用正确的data-topic属性和HTML结构
- **完整的事件触发**：确保所有必要的input和change事件被触发

## 使用说明

### 正常使用
1. 插件会自动打开发布页面：`https://creator.xiaohongshu.com/publish/publish?source=official&from=tab_switch`
2. 自动点击"图文"按钮切换到图文发布模式
3. 自动尝试多个选择器找到上传元素
4. 找到可用元素后会在控制台显示使用的选择器
5. 上传成功后会显示成功消息

### 调试模式
1. 打开小红书创作者后台页面：`https://creator.xiaohongshu.com/publish/publish?from=menu&target=image`
2. 点击插件中的"调试上传元素"按钮
3. 查看日志输出，了解当前页面的元素状态
4. 检查图文按钮和上传元素的可用性

## 技术细节

### 选择器优先级
1. **最新选择器**：基于当前页面结构的精确选择器
2. **兼容选择器**：保留原有选择器以兼容旧版本
3. **属性选择器**：基于元素属性的选择器，更稳定
4. **通用选择器**：基于元素类型的通用选择器

### 错误恢复机制
- 如果所有选择器都失败，会抛出详细的错误信息
- 错误信息包含尝试的所有选择器列表
- 便于快速定位和修复问题

## 维护说明

如果将来小红书再次更改页面结构，只需要：
1. 使用调试功能获取新的选择器
2. 将新选择器添加到 `selectors` 数组的首位
3. 保留旧选择器以确保向后兼容

这种设计确保了插件的长期稳定性和可维护性。
